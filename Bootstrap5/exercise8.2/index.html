Bootstrap Form-Floating Component

The form-floating component in Bootstrap 5 creates a sleek, modern form input style where the label floats above the input field when the field has content or is focused.

Basic Form-Floating Structure

The basic structure requires the form-floating class on a wrapper div, with the input element placed before the label:

<div class="form-floating"> <input type="email" class="form-control" id="floatingEmail" placeholder="name@example.com"> <label for="floatingEmail">Email address</label> </div>

Important: The placeholder is required for the floating label to work properly, even though it won't be visible to users.

How Form-Floating Works

The floating label component works through a combination of CSS positioning and the :placeholder-shown pseudo-element:

  1. When the input is empty and not focused, the label appears to be a placeholder inside the input.
  2. When the input receives focus or contains text, the label "floats" up above the input field.
  3. This creates a clean, intuitive interface that maintains context even when the user is typing.

Supported Form Controls

Form-floating works with the following form controls:

  • <input> elements of most types (text, email, password, etc.)
  • <select> elements
  • <textarea> elements

Example with textarea:

<div class="form-floating"> <textarea class="form-control" id="floatingTextarea" placeholder="Leave a comment here"></textarea> <label for="floatingTextarea">Comments</label> </div>

Form-Floating with Select

You can also use form-floating with select elements:

<div class="form-floating"> <select class="form-select" id="floatingSelect" aria-label="Floating label select example"> <option selected>Open this select menu</option> <option value="1">One</option> <option value="2">Two</option> <option value="3">Three</option> </select> <label for="floatingSelect">Works with selects</label> </div>

Accessibility Considerations

While form-floating creates a visually appealing interface, keep these accessibility considerations in mind:

  • Always ensure proper label-input associations using the for and id attributes.
  • The placeholder text is not visible to users when the input has content, so don't rely on it for critical instructions.
  • Consider adding aria-describedby for any additional form instructions.
  • Test your forms with keyboard navigation and screen readers.

Best Practices

  • Use clear, concise labels that describe the expected input.
  • Maintain consistent spacing between form-floating elements (Bootstrap's mb-3 utility works well).
  • Combine with validation states and feedback when appropriate.
  • For complex forms, consider grouping related floating inputs together with fieldsets and legends.
