Bootstrap Column Reordering

This example demonstrates how to reorder columns on different screen sizes using order-* classes.

1. Basic Order Classes

The order-* classes allow you to change the visual order of columns without changing their order in the HTML. The default order value is 0, and you can use values from 1 to 5.

Default Order (No Classes)

1
First in HTML

This column appears first in the HTML and displays first by default.

2
Second in HTML

This column appears second in the HTML and displays second by default.

3
Third in HTML

This column appears third in the HTML and displays third by default.

<div class="row"> <div class="col-md-4">First in HTML</div> <div class="col-md-4">Second in HTML</div> <div class="col-md-4">Third in HTML</div> </div>

Reordered with Order Classes

3
First in HTML

This column appears first in the HTML but displays third with order-3.

1
Second in HTML

This column appears second in the HTML but displays first with order-1.

2
Third in HTML

This column appears third in the HTML but displays second with order-2.

<div class="row"> <div class="col-md-4 order-3">First in HTML</div> <div class="col-md-4 order-1">Second in HTML</div> <div class="col-md-4 order-2">Third in HTML</div> </div>

2. Responsive Order Classes

Bootstrap allows you to apply different order values at different breakpoints using classes like order-md-1, order-lg-2, etc.

Responsive Reordering

In the example below, the columns have different orders on different screen sizes:

  • On mobile (default): Columns appear in their HTML order (1-2-3)
  • On medium screens (≥768px): Columns appear in reverse order (3-2-1)
  • On large screens (≥992px): Columns appear in a custom order (2-1-3)

Resize your browser window to see the changes!

First Column

On mobile: Order 1 (default)

On medium: Order 3 (order-md-3)

On large: Order 2 (order-lg-2)

Second Column

On mobile: Order 2 (default)

On medium: Order 2 (order-md-2)

On large: Order 1 (order-lg-1)

Third Column

On mobile: Order 3 (default)

On medium: Order 1 (order-md-1)

On large: Order 3 (order-lg-3)

<div class="row"> <div class="col-md-4 order-md-3 order-lg-2">First Column</div> <div class="col-md-4 order-md-2 order-lg-1">Second Column</div> <div class="col-md-4 order-md-1 order-lg-3">Third Column</div> </div>

3. Practical Use Cases

Image-Text Layout Reordering

A common use case is to have an image on the left and text on the right on desktop, but stack them with the image on top on mobile.

Desktop View (≥768px)

Image on the Left, Text on the Right

On desktop screens, the image appears on the left and the text appears on the right. This is the default layout for desktop view.

Mobile View (<768px)

Image on Top, Text Below

On mobile screens, the columns stack vertically with the image on top and the text below. This is the default stacking behavior.

Alternating Image-Text Sections

For alternating image-text sections, we can use order classes to create a zigzag layout on desktop while maintaining a consistent image-first approach on mobile.

First Section

This section has the image on the left and text on the right on desktop. On mobile, the image is on top and the text is below.

Second Section (Reordered)

This section has the text on the left and image on the right on desktop, creating an alternating pattern. On mobile, the image is still on top.

Note: We've used order-md-2 for the image and order-md-1 for the text to reorder them on medium screens and larger.

<!-- Section 1 (Default Order) --> <div class="row align-items-center"> <div class="col-md-6"> <img src="image.jpg" class="img-fluid"> </div> <div class="col-md-6"> <h3>First Section</h3> <p>Content here...</p> </div> </div> <!-- Section 2 (Reordered) --> <div class="row align-items-center"> <div class="col-md-6 order-md-2"> <img src="image.jpg" class="img-fluid"> </div> <div class="col-md-6 order-md-1"> <h3>Second Section</h3> <p>Content here...</p> </div> </div>

4. First-Last Reordering

Another common pattern is to have certain elements appear first on mobile but last on desktop (or vice versa).

Example: Call-to-Action Button

In this example, we want the call-to-action button to appear at the top on mobile (for visibility) but at the bottom on desktop (after the content).

Desktop View (≥768px)

Product Information

This is the main product information that appears first on desktop.

Lorem ipsum dolor sit amet, consectetur adipiscing elit. Nullam in dui mauris. Vivamus hendrerit arcu sed erat molestie vehicula.

Mobile View (<768px)

Product Information

This is the main product information that appears second on mobile.

Lorem ipsum dolor sit amet, consectetur adipiscing elit. Nullam in dui mauris. Vivamus hendrerit arcu sed erat molestie vehicula.

<div class="row"> <div class="col-md-12 order-2 order-md-1"> <h3>Product Information</h3> <p>This is the main product information...</p> </div> <div class="col-md-12 order-1 order-md-2 mb-3"> <button class="btn btn-success btn-lg">Buy Now</button> </div> </div>
