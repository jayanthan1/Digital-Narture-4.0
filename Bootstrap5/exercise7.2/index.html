1. Basic Text Transformations

Bootstrap provides three utilities for text transformation that change how text is capitalized.

<p class="text-uppercase">This text is transformed to uppercase.</p> <p class="text-lowercase">THIS TEXT IS TRANSFORMED TO LOWERCASE.</p> <p class="text-capitalize">this text is capitalized (first letter of each word).</p>
How Text Transformations Work
  • text-uppercase: Transforms all characters to uppercase (equivalent to CSS text-transform: uppercase;)
  • text-lowercase: Transforms all characters to lowercase (equivalent to CSS text-transform: lowercase;)
  • text-capitalize: Capitalizes the first character of each word (equivalent to CSS text-transform: capitalize;)

7. Responsive Text Transformation

While Bootstrap doesn't provide built-in responsive text transformation utilities, you can create your own with custom CSS or use media queries with the existing utilities.

This text will be uppercase only on medium screens and larger.

Note: This example uses custom CSS to create a responsive text transformation utility. Bootstrap doesn't include responsive variants for text transformation utilities by default.

<style> /* Custom responsive text transformation */ @media (min-width: 768px) { .text-md-uppercase { text-transform: uppercase !important; } } </style> <p class="text-md-uppercase">This text will be uppercase only on medium screens and larger.</p>

8. Best Practices for Text Transformation

When to Use Text Transformation

Text transformation utilities should be used thoughtfully to enhance readability and visual hierarchy. Here are some guidelines:

  • Use text-uppercase for: Headings, buttons, navigation items, labels, and badges where you want to create emphasis or visual distinction.
  • Use text-lowercase for: Stylistic purposes or to normalize text that might be entered in different cases.
  • Use text-capitalize for: Titles, names, and places where proper capitalization is needed but the source text might not have it.

Remember that excessive use of uppercase text can make content harder to read and may appear as "shouting" in digital communication.

Accessibility Considerations

When using text transformations, keep these accessibility considerations in mind:

  • All-uppercase text can be more difficult to read, especially for users with certain cognitive disabilities.
  • Screen readers read text as it appears in the HTML, not as it's visually styled with CSS.
  • For important text that needs to be properly capitalized for screen readers, ensure the actual HTML content has proper capitalization, not just the visual styling.
